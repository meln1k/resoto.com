Access Permissions

To use Resoto with all it's features‚Äîincluding cleanup‚Äîit requires wide permissions. It needs to collect all the resources, do tag validations and updates as well as deleting resources.

To use Resoto in a read-only capacity you can limit access to your cloud provider accordingly.

info

Resoto will NOT delete resources marked for deletion by default, even with the neccessary permissions.

Resoto will silently ignore collecting specific resources if it does not have the required permissions.

AWS Permissions‚Äã

To enable the full capabilities of Resoto it needs a PowerUser like role and then deny the ones we don't want it to have.

Full capabilities‚Äã

For production use‚Äîwhere Resoto also clean-up and validate tags‚Äîthe needed permission role is a PowerUser that is locked down to not be able to create any resources or modify IAM settings.

The following three policies are needed for Resoto to run with full capabilities.

Resoto allow
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "ResotoAllow",
      "Effect": "Allow",
      "Action": "*",
      "Resource": "*"
    }
  ]
}
